# GitHub Setup Checklist

Use this checklist to finish the GitHub-side baseline for QuickSnap.

## Repository settings

- [ ] Confirm the default branch is `main`.
- [ ] Add a repository description, homepage, and topics.
- [ ] Decide whether the repository should be public or private during early releases.
- [ ] Enable automatic deletion of head branches after merge.

## Branch protection

- [ ] Protect `main`.
- [ ] Require pull requests before merge.
- [ ] Require the `build` status check from GitHub Actions.
- [ ] Block direct pushes to `main`.
- [ ] Block force pushes to `main`.
- [ ] Require branches to be up to date before merge if the team prefers stricter gating.

## Actions and automation

- [ ] Enable GitHub Actions for the repository.
- [ ] Confirm `.github/workflows/swift.yml` runs on pull requests and pushes to `main`.
- [ ] Confirm `.github/workflows/release.yml` runs when a `v<version>` tag is pushed.
- [ ] Confirm CI passes on a sample pull request.
- [ ] Enable Dependabot version updates.
- [ ] Enable Dependabot security alerts.
- [ ] Enable secret scanning if available for the repository plan.

## Release operations

- [ ] Use tags in the format `v<version>`.
- [ ] Keep `VERSION` aligned with the pushed release tag.
- [ ] Add a matching `CHANGELOG.md` section before pushing the release tag.
- [ ] Confirm the tag-triggered release workflow publishes the zip generated by `./scripts/package_release.sh`.
- [ ] Restrict release publishing permissions to trusted maintainers only.
- [ ] Decide whether notarized release assets will continue to be uploaded manually.

## Contribution flow

- [ ] Confirm the PR template appears for new pull requests.
- [ ] Confirm the bug template appears for new issues.
- [ ] Confirm security issues are routed to the private email contact.
- [ ] Decide whether to add `CODEOWNERS` once the maintainer list is stable.

## Optional next steps

- [ ] Add CI artifact uploads if they become useful for testing.
- [ ] Add Discussions only if community traffic starts to justify it.
